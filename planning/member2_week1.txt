WEEK 1 AIM: High-Speed Foundation & Compute Engine (API Routing & Business Logic)

ROLE: You are the "Bridge" of MiniCloud. Your primary goal this week is to create the REST API endpoints that the Frontend (Member 3) will call, and route those calls to the Core Engine (Member 1).

---

STEP 1: DEFINE THE API CONTRACTS
- Before coding, talk to Member 3 (Frontend) and agree on exactly what the JSON requests and responses will look like.
- Example Login Request: `POST /api/auth/login` with `{"username": "admin", "password": "password123"}`
- Example Login Response: `{"token": "eyJhb...", "message": "Login successful"}`
- Example Compute Request: `POST /api/compute/launch`
- Example Compute Response: `{"instanceId": "a1b2c3d4", "status": "RUNNING"}`

STEP 2: BUILD THE AUTHENTICATION CONTROLLER
- Wait for Member 1 to finish setting up the `UserRepository` and `JwtUtil`.
- Create an `AuthController` annotated with `@RestController`.
- Implement `POST /api/auth/register`:
  - Hash the incoming password using Spring Security's `PasswordEncoder`.
  - Save the new `User` to the database using Member 1's repository.
- Implement `POST /api/auth/login`:
  - Verify the username and password against the database.
  - If valid, use Member 1's `JwtUtil` to generate a token and return it in the JSON response.
- *Dependency checkpoint*: Member 3 needs these auth endpoints running so they can build the JavaFX Login Screen.

STEP 3: BUILD THE COMPUTE CONTROLLER
- Create a `ComputeController` annotated with `@RestController` and mapped to `/api/compute`.
- Wait for Member 1 to finish the `DockerService`.
- Implement `POST /api/compute/launch`.
- In this endpoint:
  1. Extract the user details from the incoming JWT token (to know WHO is launching the instance).
  2. Call Member 1's `DockerService.launchTomcatContainer()`.
  3. Take the returned Container ID and format it into a clean JSON response.
  4. Return HTTP 200 OK.
- *Dependency checkpoint*: Member 3 will bind their "Launch Instance" button to this endpoint.

STEP 4: RIGOROUS API TESTING (POSTMAN)
- You are responsible for ensuring the "Bridge" is structurally sound before Member 3 drives over it.
- Open Postman.
- Hit `/api/auth/register` to create a test user.
- Hit `/api/auth/login` to get a JWT token.
- Copy that JWT token into the "Authorization: Bearer <token>" header in Postman.
- Hit `/api/compute/launch`.
- Verify a 200 OK response and physically check Docker Desktop to see if the container started.

END OF WEEK 1 CHECKLIST FOR MEMBER 2:
[ ] API Contracts agreed upon with Member 3.
[ ] `POST /api/auth/login` correctly returns a JWT token.
[ ] `POST /api/compute/launch` is secured, requires a token, and successfully triggers Member 1's Docker code.
[ ] All endpoints fully tested in Postman.
