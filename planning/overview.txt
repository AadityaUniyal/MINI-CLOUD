# Java Cloud Service PBL - 2 Month Pure Java Implementation Plan (8-Week High-Speed Plan)

## Project Overview

**Project Name**: MiniCloud - Java-Based Cloud Infrastructure Platform

**Concept**: A simplified cloud infrastructure platform built entirely with Java.
- **Compute Service (EC2-like)**: Create, manage, and monitor virtual instances via Docker.
- **Identity & Access Management (IAM)**: User roles, permissions, and secure access control.
- **Object Storage (S3-like)**: File storage using Java NIO for bucket management.
- **Relational Database Service (RDS-like)**: On-demand provisioning of MySQL/PostgreSQL instances.
- **Load Balancer (ELB-like)**: Pure Java reverse proxy to distribute traffic across Compute instances.

**Team Size**: 3 Members  
**Duration**: 8 Weeks (2 Months)  
**Milestone**: 70% Functional & Integrated by the end of Week 3.

---

## Tech Stack (Pure Java "Best Options" 2026)

### Frontend Layer
- **Technology**: **JavaFX** (Hardware Accelerated, Modern UI)
- **UI Framework**: JavaFX Controls + CSS Styling (Dark Mode)
- **HTTP Client**: Java 11+ `HttpClient`

### Backend Layer
- **Framework**: **Spring Boot 3.x**
- **Web Layer**: Spring Web MVC (REST Controllers)
- **Load Balancer**: Spring Cloud Gateway / Pure Java Reverse Proxy
- **Security**: Spring Security (JWT)
- **Container Simulation**: Docker Java API (for EC2 and RDS)

### Database & Infrastructure Layer
- **Platform DB**: **H2 Database** (Embedded Pure Java SQL DB)
- **Container Runtime**: **Docker** (to simulate EC2/RDS instances)
- **Storage**: Local file system (Java NIO)

---

## System Architecture

When a user interacts with the **JavaFX Desktop App**, their actions trigger an event in the UI controller. This controller uses the Java `HttpClient` to send a secure REST API request to the **Spring Boot Backend**.

The Backend's **API Gateway** routes it to the appropriate Service:
1. **Compute Service**: Commands the local Docker Engine via `docker-java` to spin up Tomcat containers (EC2-like).
2. **Storage Service**: Writes files directly to a designated folder using Java NIO (S3-like).
3. **Database Service**: Provisions MySQL/Postgres Docker containers and generates JDBC connection strings (RDS-like).
4. **Load Balancer Service**: Configures a Spring Cloud Gateway dynamically to route traffic across multiple running Compute containers.

## 8-Week Accelerated Timeline (70% Functional in 3 Weeks)

### Phase 1: High-Speed Core Functionality (Weeks 1-3)
**Target: 70% of the platform's features functionally working and integrated.**

#### Week 1: Foundation, IAM & Compute Engine
- Initialize Spring Boot, H2 DB, Spring Security, and initial Docker-Java integration.
- Build Auth endpoints and basic APIs to command Docker to launch instances.
- Setup JavaFX, Login Screen, and basic Dashboard layout.

#### Week 2: Storage Service & Database Service
- Setup Java NIO directory structures for storage buckets.
- Implement Docker logic to spin up stateful MySQL/Postgres containers.
- Build Storage UI with drag-and-drop, and Databases tab for 1-click DB provisioning.

#### Week 3: Load Balancing & Full Integration (The 70% Milestone)
- Map Load Balancers to Compute instances in the H2 Database.
- Implement the Java reverse proxy / Spring Cloud Gateway logic to route traffic.
- Ensure all JavaFX UI tabs (Compute, Storage, DB, Network) seamlessly talk to Backend. By the end of this week, the core cloud platform is fundamentally usable.

### Phase 2: Monitoring, Polish & Testing (Weeks 4-8)

#### Weeks 4-5: Real-Time Monitoring & Billing
- **Backend**: `@Scheduled` tasks to poll CPU/RAM from Docker, SSE APIs, and Bill calculation logic.
- **Frontend**: JavaFX `Timeline` threads to render Live LineCharts and Cost Dashboard.

#### Weeks 6-7: Premium Polish
- **Backend**: Global Exception Handlers, input validation, H2 DB Indexing, Audit Logging.
- **Frontend**: Animations, transitions, Dark Mode CSS theme, polished error dialogs.

#### Week 8: Packaging & Finalization
- **Team**: JUnit/Postman testing, use `jpackage` to create standalone `.exe`/`.jar` installers, record Final Demo Video.
