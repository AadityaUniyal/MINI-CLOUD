WEEK 1 AIM: High-Speed Foundation & Compute Engine (Database & Core Architecture)

ROLE: You are the "Engine Room" of MiniCloud. Your primary goal this week is to establish the base infrastructure, secure the data, and build the core functionality that allows Java to talk to Docker.

---

STEP 1: INITIALIZE THE SPRING BOOT PROJECT
- Go to Spring Initializr (start.spring.io).
- Select Java 17+, Maven, and Spring Boot 3.x.
- Add Dependencies: Spring Web, Spring Data JPA, H2 Database, Spring Security, and Lombok.
- Generate the project and share the repository with Member 2.

STEP 2: CONFIGURE THE H2 DATABASE (PERSISTENT STATE)
- Open `src/main/resources/application.properties`.
- Configure H2 to write to a local file instead of purely in-memory. This ensures data survives when the backend restarts.
  Example property: `spring.datasource.url=jdbc:h2:file:./data/minicloud`
- Enable the H2 Console for easy debugging: `spring.h2.console.enabled=true`

STEP 3: SECURE THE PLATFORM (IAM & JWT)
- Create a `User` entity class using `@Entity`. It needs `id`, `username`, `password` (hashed), and `role`.
- Create a `UserRepository` interface extending `JpaRepository`.
- Implement Spring Security:
  - Create a `SecurityFilterChain` bean to secure `/api/**` endpoints but allow `/api/auth/**`.
  - Write a `JwtUtil` utility class that can generate, sign, and validate JSON Web Tokens.
  - Write a `CustomUserDetailsService` to load users from the H2 database during login.
- *Dependency checkpoint*: Member 2 needs your `JwtUtil` and `UserRepository` to build the login API.

STEP 4: THE COMPUTE BREAKTHROUGH (DOCKER-JAVA)
- Add the `docker-java` dependency to your `pom.xml`.
  ```xml
  <dependency>
      <groupId>com.github.docker-java</groupId>
      <artifactId>docker-java</artifactId>
      <version>3.3.x</version>
  </dependency>
  ```
- Create a `DockerService` class annotated with `@Service`.
- Write logic to build a `DockerClient` connecting to the host's Docker daemon (e.g., `tcp://localhost:2375` on Windows, or the Unix socket on Linux/Mac).
- Write your first core engine method: `public String launchTomcatContainer()`.
  - Use the DockerClient to pull the `tomcat:latest` image.
  - Create a new container command.
  - Start the container.
  - Return the generated Container ID.
- *Dependency checkpoint*: Member 2 relies on this `launchTomcatContainer()` method to hook it up to the web API.

END OF WEEK 1 CHECKLIST FOR MEMBER 1:
[ ] Spring Boot runs without errors.
[ ] H2 database file is created locally and `User` tables exist.
[ ] Spring Security intercepts unauthorized requests.
[ ] `launchTomcatContainer()` successfully spins up a real Docker container on your machine.
