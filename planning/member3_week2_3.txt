WEEK 2 & 3 AIM: Unified Cloud Console & One-Click Deployment

ROLE: You are the "UX Architect." Your mission is to make the complex MiniCloud infrastructure feel as simple and powerful as the AWS Console.

---

WEEK 2: UNIFIED CLOUD DASHBOARD

STEP 1: STORAGE & DATABASE INTERFACES
- Build the "Storage Browser" (File upload/download tables) and the "Database Manager" (Cards with connection strings).
- Ensure smooth "loading" indicators as infrastructure is being provisioned.

STEP 2: NETWORKING UI (LOAD BALANCERS)
- Design the "Networking" tab where users can create Load Balancers and add Compute instances to them visually.

---

WEEK 3: ONE-CLICK WEBSITE DEPLOYMENT

STEP 3: "WEBSITE WIZARD" TEMPLATES
- Create a special "Website Deployment" UI.
- Similar to AWS Amplify/CloudFormation, the user should be able to select a template (e.g., "Java Web App with MySQL") and click "Deploy."
- This will trigger Member 2's Orchestration API to build the entire stack.

STEP 4: HEALTH MONITORING DASHBOARD
- Build the "Operations" dashboard.
- Features: 
  - Live Gauge charts for CPU/RAM.
  - A "Timeline" showing if a container was "Auto-Healed" by the system.
  - A visual "Green/Red" status indicator for each hosted website.
