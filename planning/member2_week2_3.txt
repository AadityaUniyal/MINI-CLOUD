WEEK 2 & 3 AIM: Dynamic Routing & Orchestration Automation

ROLE: You are the "Network Lead." Your mission is to build the software-defined network that connects users to their websites effortlessly.

---

WEEK 2: THE BRIDGE & DYNAMIC ROUTER

STEP 1: STORAGE & DB APIs
- Implement secure REST controllers for direct file uploads and database provisioning.
- All requests must be validated against the user's quota.

STEP 2: THE LOAD BALANCER (REVERSE PROXY)
- Implement a Dynamic Reverse Proxy (using Spring Cloud Gateway or a custom Java Filter).
- It must intercept traffic on a public port and route it to multiple Tomcat containers using a **Round-Robin** algorithm.
- *Dependency checkpoint*: Member 3 needs this to show the Load Balancer's "Active Traffic" in the UI.

---

WEEK 3: DEPLOYMENT ENGINE (AUTOMATION)

STEP 3: "FULL-STACK" DEPLOYMENT API
- Build an orchestration endpoint: `POST /api/orchestration/deploy-stack`.
- Logic: In a single API call, it must:
  1. Provision a New Tomcat Container.
  2. Provision a New MySQL Instance.
  3. Create a Load Balancer pointing to the Tomcat container.
  4. Return the entire "Stack Information" to the user.

STEP 4: MONITORING & METRICS API
- Create endpoints to expose Docker stats (CPU/RAM) as live JSON data for the frontend.
- Support real-time health updates for the "Auto-Healing" containers.

---

END OF WEEKS 2 & 3 CHECKLIST FOR MEMBER 2:
[ ] Load Balancer successfully routes traffic between containers.
[ ] Storage and DB APIs are secured and isolated per user.
[ ] "Deploy Stack" API can launch a Web+DB+LB combo in one click.
[ ] Real-time Metrics API provides accurate data to the dashboard.
