WEEK 2 AIM: Storage Browser & Database Management UI

ROLE: You are the "Face" of MiniCloud. This week you build the interactive dashboard features that make the cloud feel real to the user.

---

STEP 1: STORAGE BROWSER UI
- Create a new FXML/Scene for "Storage".
- Features:
  - A `TableView` showing File Name, Size, and Upload Date.
  - "Upload File" Button: Triggers `FileChooser`, then sends a Multipart POST request to Member 2's API.
  - "Download" Button: Downloads the selected file and saves it to the user's local Downloads folder.
  - "Refresh" Button: Reloads the file list.

STEP 2: DATABASE PROVISIONING UI
- Create a new FXML/Scene for "Databases".
- Features:
  - "+ New Database" Button: Opens a small Dialog/Popup asking for Database Name and Root Password.
  - Instance List: Use a `VBox` or `FlowPane` to show "Database Cards".
  - Each card should show: DB Name, Status (Running/Starting), and a "Copy JDBC String" button.

STEP 3: DASHBOARD INTEGRATION
- Update your main Sidebar navigation to switch between "Compute", "Storage", and "Databases".
- Use a `StackPane` in the main center area to swap between these views without opening new windows.

END OF WEEK 2 CHECKLIST FOR MEMBER 3:
[ ] UI can display a list of files fetched from the backend.
[ ] User can select a local file and "Upload" it successfully via the UI.
[ ] User can click "New Database" and see a new entry appear in the list once provisioned.
