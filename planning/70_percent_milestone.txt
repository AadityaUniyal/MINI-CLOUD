To demonstrate 70% project completion (Phase 1), the following five core 
infrastructure services must be 100% working and ready for a demo.

1. IDENTITY & ACCESS MANAGEMENT (IAM)
-------------------------------------
- Functional: User Registration, User Login, and JWT Token Issuance.
- Requirement: All other services must be secured with a 'Bearer <JWT>' header.
- Status Check: Can you log in and get a token?

2. COMPUTE SERVICE (EC2-LIKE)
-----------------------------
- Functional: Launching Tomcat containers, Listing instances, Terminating instances.
- Requirement: Containers must start on the host Docker Engine and be accessible
  (e.g., via localhost:8080).
- Status Check: Does 'docker ps' show your new Tomcat container?

3. MANAGED DATABASE SERVICE (RDS-LIKE)
--------------------------------------
- Functional: Provisioning MySQL containers with dynamic port mapping (3306x).
- Requirement: Dynamic port allocation logic must prevent port conflicts.
- Status Check: Can you connect to the MySQL instance using a DB tool?

4. OBJECT STORAGE SERVICE (S3-LIKE)
-----------------------------------
- Functional: Creating Buckets, Multi-part file uploads, Listing bucket files.
- Requirement: Files must be stored on disk using Java NIO with logical isolation
  per user/bucket.
- Status Check: Does the file appear in './storage/{user}/{bucket}/'?

5. LOAD BALANCER (MANAGEMENT)
-----------------------------
- Functional: Creating a Load Balancer record and linking it to multiple 
  running Compute instances.
- Requirement: The system should track which instances are part of which LB pool.
- Status Check: Can you see the LB configuration in your Dashboard/API?

--------------------------------------------------------------------------------
OVERALL GOAL: A user should be able to log in, launch a server, provision a DB, 
and upload a config file without manual intervention or command-line steps.
